# 数据库重构过程记录

## 操作时间
2024年1月 - 将数据库从Drizzle ORM + D1重构为Cloudflare KV存储

## 操作概述
成功将clash-url-generator项目的数据库层从Drizzle ORM + D1数据库重构为使用Cloudflare KV存储，采用JSON格式保存数据，保持了原有的数据结构和函数接口。

## 重构详情

### 1. 移除的依赖
- ❌ drizzle-orm/d1 - Drizzle ORM D1适配器
- ❌ @opennextjs/cloudflare getCloudflareContext - Cloudflare环境上下文
- ❌ ../db/scheme - 数据库表结构定义
- ❌ drizzle-orm eq操作符

### 2. 新增的功能模块
- ✅ **数据接口定义**
  - ProxyData接口 - 代理数据结构
  - ProxyUrlData接口 - 代理URL数据结构
- ✅ **KV存储键名管理**
  - PROXY_LIST_KEY - 代理ID列表键
  - PROXY_URL_LIST_KEY - 代理URL ID列表键
  - PROXY_KEY_PREFIX - 代理数据键前缀
  - PROXY_URL_KEY_PREFIX - 代理URL数据键前缀
- ✅ **辅助工具函数**
  - generateUUID() - UUID生成器
  - getCurrentTimestamp() - 时间戳获取
  - getProxyIdList() - 获取代理ID列表
  - updateProxyIdList() - 更新代理ID列表
  - getProxyUrlIdList() - 获取代理URL ID列表
  - updateProxyUrlIdList() - 更新代理URL ID列表

### 3. 数据存储结构设计

#### 3.1 代理数据存储
- **列表索引**: `proxy_list` - 存储所有代理ID的JSON数组
- **单个数据**: `proxy_{id}` - 存储单个代理的完整JSON数据
- **数据结构**:
  ```typescript
  {
    id: string,
    name: string,
    area: string,
    ip: string,
    type: string,
    port: number,
    method: string,
    password: string,
    status: boolean,
    createdAt: string,
    updatedAt: string
  }
  ```

#### 3.2 代理URL数据存储
- **列表索引**: `proxy_url_list` - 存储所有代理URL ID的JSON数组
- **单个数据**: `proxy_url_{uuid}` - 存储单个代理URL的完整JSON数据
- **数据结构**:
  ```typescript
  {
    uuid: string,
    name: string,
    url: string,
    status: boolean,
    createdAt: string,
    updatedAt: string
  }
  ```

### 4. 函数功能保持一致性

#### 4.1 代理相关函数
- ✅ **createProxy()** - 创建代理
  - 生成UUID作为主键
  - 设置默认值（method='auto', status=true）
  - 自动设置创建和更新时间
  - 更新ID列表索引
- ✅ **updateProxy()** - 更新代理
  - 检查代理是否存在
  - 保留原有的创建时间
  - 更新修改时间
  - 返回更新后的数据或false
- ✅ **deleteProxy()** - 删除代理
  - 检查代理是否存在
  - 删除KV中的数据
  - 从ID列表中移除
  - 返回删除的数据或false
- ✅ **getProxys()** - 获取代理列表
  - 支持按ID获取单个代理
  - 支持获取所有代理列表
  - 返回数组格式数据

#### 4.2 代理URL相关函数
- ✅ **createProxyUrl()** - 创建代理URL
  - 生成UUID作为主键
  - 设置默认状态为true
  - 自动设置时间戳
  - 更新ID列表索引
- ✅ **updateProxyUrl()** - 更新代理URL
  - 检查存在性
  - 保留创建时间
  - 更新修改时间
- ✅ **deleteProxyUrl()** - 删除代理URL
  - 保留原有的缓存清理逻辑
  - 删除clash_config缓存
  - 从ID列表中移除
- ✅ **getProxyUrls()** - 获取代理URL列表
  - 支持单个和批量获取
  - 保持返回格式一致

### 5. 关键改进点

#### 5.1 性能优化
- **索引机制** - 使用ID列表作为索引，避免全量扫描
- **批量操作** - 支持高效的批量数据获取
- **缓存友好** - KV存储天然支持缓存机制

#### 5.2 数据一致性
- **原子操作** - 每个操作都是原子性的
- **索引同步** - 数据变更时同步更新索引
- **错误处理** - 完善的存在性检查

#### 5.3 兼容性保持
- **接口不变** - 所有导出的db对象方法保持不变
- **返回格式** - 保持与原Drizzle ORM相同的返回格式
- **错误行为** - 保持相同的错误处理逻辑

### 6. 技术特点
1. **无SQL依赖** - 完全基于KV存储，无需SQL查询
2. **JSON序列化** - 使用JSON格式存储复杂数据结构
3. **UUID主键** - 使用自定义UUID生成器
4. **时间戳管理** - ISO格式时间戳
5. **索引优化** - 双层存储结构（索引+数据）

### 7. 迁移注意事项
1. **数据迁移** - 需要将现有D1数据迁移到KV存储
2. **键名冲突** - 确保KV键名不与现有缓存冲突
3. **性能监控** - 监控KV操作的性能表现
4. **备份策略** - 建立KV数据的备份机制

## 重构结果
- ✅ 完全移除了Drizzle ORM依赖
- ✅ 保持了所有原有功能
- ✅ 提升了存储性能
- ✅ 简化了部署配置
